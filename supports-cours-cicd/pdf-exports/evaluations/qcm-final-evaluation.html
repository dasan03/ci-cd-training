Évaluation finale des compétences acquises durant la formation CI/CD de 5 jours

Questions



Question 1



Quelle est la différence principale entre l'intégration continue (CI) et le déploiement continu (CD) ?

A. CI concerne les tests, CD concerne le déploiement automatisé
B. CI est manuel, CD est automatique
C. CI utilise Docker, CD utilise Kubernetes
D. Il n'y a aucune différence

Explication: L'intégration continue (CI) se concentre sur l'intégration fréquente du code et l'exécution automatique des tests, tandis que le déploiement continu (CD) automatise le processus de déploiement en production.

---

Question 2



GitHub Actions permet uniquement d'exécuter des tests, pas de déployer des applications.

Explication: GitHub Actions est une plateforme complète de CI/CD qui permet à la fois d'exécuter des tests, de construire des applications et de les déployer sur différents environnements.

---

Question 3



Quel est l'avantage principal de l'utilisation de Docker dans un pipeline CI/CD ?

A. Réduction des coûts de développement
B. Isolation et reproductibilité des environnements
C. Amélioration des performances de l'application
D. Simplification du code source

Explication: Docker permet de créer des environnements isolés et reproductibles, garantissant que l'application fonctionne de manière identique en développement, test et production.

---

Question 4



Associez chaque type de test à sa position dans la pyramide des tests :

Explication: La pyramide des tests recommande de nombreux tests unitaires à la base, un nombre modéré de tests d'intégration au milieu, et peu de tests E2E au sommet.

---

Question 5



Quelle pratique est recommandée pour optimiser les temps d'exécution des tests dans un pipeline CI/CD ?

A. Exécuter tous les tests séquentiellement
B. Paralléliser l'exécution des tests
C. Réduire le nombre de tests
D. Exécuter les tests uniquement en production

Explication: La parallélisation des tests permet de réduire significativement le temps total d'exécution du pipeline en utilisant plusieurs processeurs ou machines simultanément.
