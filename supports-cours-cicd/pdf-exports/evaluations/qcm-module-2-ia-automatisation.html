Questions



Question 1



Quel est l'avantage principal des Smart Locators dans les outils de test IA comme Testim ?

A. Ils utilisent plusieurs stratégies de localisation pour améliorer la robustesse des tests
B. Ils permettent d'exécuter les tests plus rapidement
C. Ils réduisent la consommation de ressources système
D. Ils génèrent automatiquement des rapports de test

Explication: Les Smart Locators utilisent plusieurs stratégies (texte, position, attributs, similarité visuelle) pour localiser les éléments, ce qui rend les tests plus robustes face aux changements d'interface. Si une stratégie échoue, les autres peuvent prendre le relais.

---

Question 2



L'auto-healing dans les tests automatisés peut masquer de vrais problèmes d'interface utilisateur.

Explication: Vrai. L'auto-healing peut parfois réparer automatiquement des sélecteurs cassés qui indiquent en réalité des changements importants dans l'interface. Il est crucial de surveiller les événements d'auto-healing et de valider qu'ils ne masquent pas des régressions réelles.

---

Question 3



Dans le contexte des tests visuels avec IA, que signifie 'baseline management' ?

A. La gestion des images de référence pour la comparaison visuelle
B. La configuration des seuils de performance des tests
C. L'optimisation de la vitesse d'exécution des tests
D. La gestion des versions du code de test

Explication: Le baseline management consiste à gérer les images de référence (baselines) utilisées pour comparer l'état actuel de l'interface avec l'état attendu. Ces baselines doivent être mises à jour lorsque des changements visuels légitimes sont apportés.

Question 5



Quel algorithme est couramment utilisé pour la détection d'anomalies dans les logs d'application ?

A. Linear Regression
B. Isolation Forest
C. Decision Tree
D. K-Means Clustering

Explication: Isolation Forest est particulièrement efficace pour la détection d'anomalies car il isole les points anormaux en construisant des arbres de décision aléatoires. Les anomalies nécessitent moins de divisions pour être isolées que les points normaux.

---

Question 6



Les outils de test IA peuvent complètement éliminer le besoin de maintenance des tests automatisés.

Explication: Faux. Bien que les outils IA réduisent significativement la maintenance (auto-healing, smart locators), ils ne l'éliminent pas complètement. Il faut toujours surveiller les réparations automatiques, mettre à jour les baselines visuelles, et ajuster les modèles selon l'évolution de l'application.

---

Question 7



Dans un système de détection d'anomalies basé sur l'IA, que représente le 'contamination rate' ?

A. Le pourcentage d'erreurs acceptables dans les prédictions
B. La proportion estimée d'anomalies dans les données d'entraînement
C. Le taux de faux positifs généré par le modèle
D. La vitesse de traitement des données en temps réel

Explication: Le contamination rate représente la proportion estimée d'anomalies dans le dataset d'entraînement. Ce paramètre aide l'algorithme à calibrer sa sensibilité : un taux plus élevé rend le modèle moins sensible aux anomalies.

---

Question 8



Quel est le principal défi lors de l'intégration d'outils de test IA dans un pipeline CI/CD ?

A. La configuration des tokens d'authentification
B. La gestion des résultats non-déterministes et des faux positifs
C. L'installation des dépendances logicielles
D. La compatibilité avec les navigateurs web

Explication: Le principal défi est la gestion des résultats non-déterministes. Les outils IA peuvent parfois produire des résultats différents pour les mêmes conditions, générant des faux positifs qui peuvent bloquer le pipeline. Il faut configurer des seuils appropriés et des mécanismes de validation.

---

Question 9



Les tests visuels automatisés avec IA sont généralement plus rapides que les tests fonctionnels traditionnels.

Explication: Faux. Les tests visuels sont généralement plus lents car ils nécessitent de capturer des screenshots, de les traiter avec des algorithmes de computer vision, et de les comparer avec les baselines. Ils sont cependant plus efficaces pour détecter les régressions visuelles.

---

Question 10



Quelle métrique est la plus importante pour évaluer l'efficacité d'un système d'auto-healing ?

A. Le nombre total de tests exécutés
B. Le taux de succès des réparations automatiques
C. La vitesse d'exécution des tests
D. Le nombre de développeurs utilisant l'outil

Explication: Le taux de succès des réparations automatiques (pourcentage de réparations qui corrigent effectivement le problème sans introduire de faux positifs) est la métrique clé pour évaluer l'efficacité de l'auto-healing. Un taux élevé indique que le système répare correctement les tests cassés.
